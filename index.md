# PumpUp

<p class="subtitle">The platform where business stakeholders vibe-code enterprise tools.</p>

Coding agents are changing who builds software inside companies. The person who needs the tool — the PM, the sales lead, the ops manager — is increasingly the person who can build it.

PumpUp is a full-stack platform for building, running, and governing this new class of internal application.

---

## The shift

When AI coding tools drop the fixed cost of starting something, more greenfield services get written, many more people participate in building software, and the line between prototype and production blurs.

These aren't core revenue systems. They look more like small internal services, lightweight UIs over existing data, workflow automation, and glue code between systems.

Historically, much of this lived in low-code tools or never got built at all. With coding agents, teams increasingly choose real code instead.

## The problem

These applications create immediate friction: Where do they live? How are they hosted and secured? Who owns them over time?

They don't fit the traditional software lifecycle — GitHub repos, bespoke CI/CD, long-term service ownership, platform team involvement for every deployment. That process is too slow and expensive for this category of work. But letting teams deploy ad hoc creates security, reliability, and governance problems.

## What PumpUp is

A web-based agentic builder backed by an opinionated application framework and a hosted Kubernetes runtime.

- **Web-based builder.** Stakeholders create a new app, enter a chat-first interface, and describe what they want. A coding agent builds it through conversation. A live preview updates in real-time. The stakeholder never sees the underlying code.

- **Opinionated framework.** Java backend, React frontend, typed end-to-end. Middleware, auth, state management, and integrations abstracted. When engineering needs to take over, it's real code on a blessed stack — not a proprietary abstraction.

- **Hosted runtime.** Kubernetes, integrated with enterprise identity and networking. Platform-team approved, but intentionally separate from the core SDLC lane.

- **Agent-agnostic.** PumpUp supports the Agent Client Protocol — any compatible coding agent works. Enterprises bring their own.

## Who it's for

**Primary user:** Business stakeholders who need internal tools and can describe what they want but cannot write code.

**Secondary user:** Developers who inherit PumpUp apps when they need to be extended or maintained. They get a standard project in a git repo.

**Buyer:** Platform and engineering leaders who want a sanctioned path for AI-built tools.
